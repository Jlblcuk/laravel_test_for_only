# Структура базы данных

Ниже представлена ER-диаграмма физической модели базы данных проекта.

![ER-диаграмма структуры БД](docs/img/1.png)

## Описание связей

- **`users`**
    - Принадлежит одной `position` 
    - Имеет много `trips`

- **`positions` <-> `comfort_categories`**
    - Связь многие-ко-многим через таблицу `position_comfort_category`
    - Составной первичный ключ: (`position_id`, `comfort_category_id`)

- **`cars`**
    - Принадлежит одной `comfort_category`
    - Принадлежит одному `driver`
    - Может быть забронирован во многих `trips`

- **`trips`**
    - Принадлежит одному `user`
    - Принадлежит одному `car`

- **`drivers`**
    - Имеет много `cars`
